# Reviewing & Saving

Before your mapping goes into effect, TokenNexus shows you a full coverage summary - how many variables are mapped, how many are still unmapped, and whether any mappings have type warnings. Saving commits this mapping to Figma's plugin storage so it persists across sessions and is available to anyone who opens the plugin in the same file.

<ImagePlaceholder label="Screenshot: Mapping review panel showing coverage summary with mapped vs unmapped counts" ratio="16/9" />

## The coverage summary

The summary panel shows three numbers: total variables in the file, variables with confirmed mappings, and variables with no mapping. Unmapped variables will never appear in sync diffs - they are invisible to the sync engine. A coverage gap is not an error, but it is worth reviewing before your first sync.

## Type warnings

If any confirmed mapping pairs a Figma variable to a token of a different type - for example, a number variable mapped to a color token - the summary flags it with a warning count. You can sync with type warnings present, but the applied value may be incorrect. Resolve them in the mapping panel before syncing.

<Callout variant="warning">
  Saving overwrites the previously saved mapping for this Figma file. There is no versioned mapping history within the plugin. Use the Export feature before making large changes if you want to be able to restore a previous mapping state.
</Callout>
