# Manual Mapping

Manual mapping lets you create a direct, explicit connection between any Figma variable and any token in your selected files. Use it to handle the cases that auto-match cannot resolve - naming convention divergence, ambiguous duplicates, or tokens that are intentionally named differently from their corresponding variables.

<ImagePlaceholder label="Screenshot: Manual mapping panel with variable picker on left and token search on right" ratio="16/9" />

<StepBlock number={1} title="Open the Mapping panel">
  In the TokenNexus panel, navigate to the Mapping tab. Unmapped variables are listed at the top under "Needs mapping." Click any variable to begin mapping it.
</StepBlock>

<StepBlock number={2} title="Select a token">
  Use the token search field to find the token you want to map to this variable. You can search by key path, by value, or by file name. The search is fuzzy and works across all your selected token files simultaneously.
</StepBlock>

<StepBlock number={3} title="Confirm the pairing">
  Review the variable name, the token path, both current values, and their types before clicking "Confirm." A type mismatch will be flagged with a warning - you can still confirm, but TokenNexus will note the incompatibility in the mapping record.
</StepBlock>

<Callout variant="note">
  Manual mappings take priority over auto-match suggestions. Once you manually confirm a mapping for a variable, auto-match will never overwrite it - only you can change or delete it.
</Callout>
