# Auto-Match

Auto-Match is TokenNexus's automated suggestion engine. When you run it, the plugin scores every combination of Figma variable and design token using three signals - name similarity, value proximity, and type compatibility - and surfaces the most confident pairings for your review. It does not create mappings; it creates candidates. You review, correct, and confirm.

<ImagePlaceholder label="Screenshot: Auto-Match results panel with confidence scores and suggested pairings" ratio="16/9" />

<Callout variant="tip">
  Always run Auto-Match first on any new file. Even if it does not achieve 100% coverage, it will correctly resolve the bulk of straightforward pairings and leave you with a much smaller set of manual decisions to make.
</Callout>

## How the scoring works

Each candidate pairing receives a composite score between 0 and 1. The score is computed from three weighted components:

**Name similarity (weight: 0.5)** - The normalized edit distance between the token key path and the Figma variable name, after both are lowercased and delimiters (`.`, `/`, `-`, `_`) are collapsed. A token named `color.brand.primary` and a variable named `Color/Brand/Primary` score close to 1.0 on this component.

**Value proximity (weight: 0.35)** - For <TypeBadge type="COLOR" /> tokens, the CIELAB color distance between the token value and the current Figma variable value. For <TypeBadge type="FLOAT" /> tokens, the normalized numerical difference. For <TypeBadge type="STRING" /> and <TypeBadge type="BOOLEAN" /> tokens, exact match returns 1.0, mismatch returns 0.

**Type compatibility (weight: 0.15)** - A binary gate. If the token type and the Figma variable type are incompatible, the score is set to 0 regardless of name similarity.

## The confidence threshold system

<ConfidenceMeter />

Scores below **0.38** are not shown. They represent pairings where at least two of the three signals disagree significantly - showing these would add noise without adding value.

Scores between **0.38 and 0.75** appear as suggestions in the review panel. These are plausible pairings that the plugin is not confident enough to apply automatically. You should inspect each one before confirming or rejecting it.

Scores above **0.75** are treated as high-confidence matches and are pre-selected in the review panel. You can still reject them individually before committing.

## Reviewing auto-match results

<ImagePlaceholder label="Screenshot: Auto-Match review panel with accept/reject controls per suggestion" ratio="4/3" />

The review panel shows each suggestion as a row: variable name on the left, token path on the right, confidence score as a small numerical badge. Rows are sorted by descending confidence. You can accept all, reject all, or toggle individual rows before clicking "Confirm Mapping."

<Callout variant="note">
  Confirming a suggestion creates a saved mapping record. Rejecting a suggestion does not create any record - it simply signals that this pairing should not be auto-suggested again in the current session.
</Callout>
