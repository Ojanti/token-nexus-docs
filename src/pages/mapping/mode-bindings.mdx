# Mode Bindings

Figma variables support multiple modes - Light and Dark are the most common example, but a variable collection can have any number. Mode bindings let you map each mode of a Figma variable to a different token, or to the same token in a different source file. This is how you sync a complete theme system: one variable, two modes, two token sources.

<ImagePlaceholder label="Screenshot: Mode binding panel showing Light mode bound to light.json and Dark mode bound to dark.json" ratio="16/9" />

## How mode bindings work

A standard mapping connects one Figma variable to one token. A mode binding extends this to connect one Figma variable to one token per mode. Each binding is independent - you can bind `Color/Brand/Primary` in Light mode to `color.brand.primary` in `light.json`, and in Dark mode to `color.brand.primary` in `dark.json`. When you sync, each mode is updated from its bound token source.

<Callout variant="tip">
  If your token file structure mirrors your Figma variable mode structure - for example, one JSON file per theme - mode bindings are how you connect them. Set up bindings once and all subsequent syncs update all modes in a single operation.
</Callout>
