## What TokenNexus is

Design tokens live in your codebase. They are the single source of truth for color, spacing, typography, and every other design decision that spans both design and engineering. They live in JSON files, versioned in Git, reviewed in pull requests, and consumed by your build pipeline. They have names like `color.brand.primary` and values like `#3B82F6`.

Figma variables live in Figma. They power the modes, themes, and component logic that make your design files maintainable at scale. They have names like `Color/Brand/Primary` - the same concept, a different syntax, enforced by a different tool. These two worlds have always required a human in the loop to keep them in sync: someone reading a GitHub diff, then manually updating values in Figma. Someone who gets pulled away, forgets, or makes a typo.

TokenNexus removes that human bottleneck. It creates a named, persisted mapping between every Figma variable and its corresponding design token. When tokens change in your GitHub repository, TokenNexus fetches those changes, builds a diff, and presents every update for your review. You decide what to apply. When you confirm, the values are written directly to your Figma variables - no export, no import, no broken references, no intermediate file to lose track of.

## What it is NOT

Before you go further, it helps to be clear about what TokenNexus does not do.

- Not a token management tool - you manage your tokens in your codebase, exactly as you do today
- Not a design system generator - it does not create variables, collections, or components from scratch
- Not a two-way sync - changes you make to Figma variables do not push back to GitHub automatically
- Not a replacement for your token pipeline - it works alongside your existing Style Dictionary or Theo setup
- Not a hosted service - it runs entirely within the Figma plugin sandbox, with your GitHub credentials stored in your browser's local storage, never transmitted to any third-party server

## Before you install

To use TokenNexus, you need all three of the following:

<ul class="checklist">
  <li>A Figma file with local variables already defined</li>
  <li>A GitHub repository containing token JSON files</li>
  <li>A GitHub Personal Access Token with <code>repo:read</code> scope</li>
</ul>

If you are missing any of these, the [Setup section](/token-nexus-docs/setup/connecting-repository) explains exactly how to prepare each one.

<Callout variant="note">
  TokenNexus reads your design tokens from GitHub. It does not write to your repository under any circumstances - unless you explicitly use the export feature described in [Export & Import](/token-nexus-docs/advanced/export-import). Your GitHub credentials are stored locally in Figma's plugin storage and are never sent to any server other than GitHub's own API.
</Callout>
