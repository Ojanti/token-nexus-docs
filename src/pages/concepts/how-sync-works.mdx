# How Sync Works

TokenNexus never applies changes without showing them first. Sync is a review session, not a push button. The plugin fetches the latest token values from GitHub, computes a diff against the values your mapped Figma variables currently hold, and presents every change for your inspection before a single variable is touched.

<ImagePlaceholder label="Screenshot: Sync diff panel showing pending changes with old and new values" ratio="16/9" />

## The sync sequence

<StepBlock number={1} title="Fetch">
  TokenNexus calls the GitHub Contents API using your stored PAT to read the current state of your mapped token files. Only the files referenced in your mapping are fetched - not your entire repository.
</StepBlock>

<StepBlock number={2} title="Diff">
  For each mapped variable, the plugin compares the token's current `$value` against the variable's current value in Figma. Changes, additions, and removals are classified and collected into a diff set.
</StepBlock>

<StepBlock number={3} title="Review">
  The diff is presented in the Sync panel. You see old value, new value, the variable name, and the token path for every change. You can deselect individual updates before applying.
</StepBlock>

<StepBlock number={4} title="Apply">
  When you confirm, TokenNexus writes only the selected updates to your Figma variables using the Plugin API. Nothing else in your file is touched.
</StepBlock>

<Callout variant="note">
  Sync reads your Figma variables at the moment you initiate it. If another team member is editing variables in the same file simultaneously, their changes may not be reflected in the diff. Coordinate with your team before running sync on a shared file.
</Callout>

## Sections

## What triggers a sync

Sync is always manual. The plugin does not poll for changes in the background. You initiate sync from the plugin panel whenever you are ready to review and apply updates.

## Partial application

How to cherry-pick individual changes from a diff - for example, applying color updates but holding back a spacing change that needs design review first.

## What changes are not shown

Token keys that exist in your files but are not in your mapping will never appear in a sync diff. Only mapped pairs are compared.

## Sync history

How the plugin records a log of past sync sessions, including which changes were applied and which were skipped.
