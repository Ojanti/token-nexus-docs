# Quickstart

Get from a fresh plugin install to a working sync. Two paths depending on where you are starting from.

## Which path are you on?

**Path A - You already have Figma variables.** Follow all six steps below. Auto-match does most of the mapping work for you.

**Path B -You have tokens in GitHub but no Figma variables yet**. Follow steps 1 to 3, then use the Generate Variables feature inside the plugin. It creates your collections, modes, and mappings in one step. Then jump to step 6 to run your first sync.

**Path C - No Figma variables and no tokens yet.** Fork [Ojanti/mojaui-tokens](https://github.com/Ojanti/mojaui-tokens) to get a real DTCG token set you can use immediately. Not sure how to fork a repo? [GitHub's forking guide](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/working-with-forks/fork-a-repo) walks through it in two minutes. Then come back and follow Path B.

---

## Before you start

- A Figma file open (with or without variables)
- A GitHub repository containing token JSON files

---

<GuideStep step={1} title="Install TokenNexus" tip="The plugin runs in the context of the file you have open. If you switch files, reopen it from Plugins - TokenNexus.">
  Open Figma and go to the Community panel. Search for **TokenNexus** or [visit the plugin page directly](https://www.figma.com/community/plugin/1617061240636708738/tokennexus-for-figma). Click Install or Open with. Once installed, open the file you want to connect and launch the plugin from Plugins - TokenNexus.

  <ImagePlaceholder label="Screenshot: Figma Community page showing the TokenNexus plugin install button" />
</GuideStep>

<GuideStep step={2} title="Connect your GitHub repository">
  The plugin opens on a setup wizard. Enter your repository in `owner/repo` format (example: `acme/design-tokens`), your Personal Access Token, and the folder path where your token files live (example: `tokens/` or `src/tokens/`).

  <Callout variant="tip">
    Not sure where to get a GitHub Personal Access Token? [GitHub's guide to creating a personal access token](https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/managing-your-personal-access-tokens) walks through it step by step. You need read access to the repository - nothing more.
  </Callout>

  <ImagePlaceholder label="Screenshot: TokenNexus setup wizard showing the repo connection form" />
</GuideStep>

<GuideStep step={3} title="Select your token files" tip="If no files appear, the folder path is wrong. Check your repository structure and make sure the path matches exactly, including any trailing slashes.">
  TokenNexus scans the folder and lists every JSON file it finds. Select the files that are your token sources. Skip generated output files, config files, or anything that is not a token definition.

  <ImagePlaceholder label="Screenshot: Token file selection screen showing a list of JSON files" />
</GuideStep>

<Callout variant="note">
  **Path B users:** if you have no variables in Figma yet, you will see a Generate Variables option after file selection. Use it - TokenNexus reads your tokens and creates the variable collections, modes, and mappings for you. Skip to step 6 when done.
</Callout>

<GuideStep step={4} title="Run auto-match" tip="A common reason for a low match rate is a big naming difference between your Figma file and your token file. A quick manual pass on the suggestions list usually catches the rest.">
  In the mapping panel, click **Auto-Match**. TokenNexus scores every possible variable-to-token pairing using names, values, types, and path structure. High-confidence matches (above 0.75) apply automatically. Lower-confidence matches appear as suggestions for you to confirm or dismiss.

  On a well-structured file, expect 70 to 90 percent of variables matched before you touch anything.

  <ImagePlaceholder label="Screenshot: Mapping panel after auto-match - showing matched variables, suggestions, and unmatched list" />
</GuideStep>

<GuideStep step={5} title="Review suggestions and map the rest">
  Open the suggestions panel. Each suggestion shows the Figma variable, the proposed token, and the current values from both sides. Confirm the ones that look right. Dismiss the wrong ones and manually assign them from the token picker.

  For any unmatched variable, click its row and search for the token by name or value.

  <Callout variant="warning">
    **Save before you move on.** Mappings are held in memory as you work - they are not written to the file until you click Save. If you close the plugin or the file before saving, your work is gone. Click **Save** in the top bar now.
  </Callout>

  <ImagePlaceholder label="Screenshot: Suggestions panel with confirm and dismiss actions on each suggestion" />
</GuideStep>

<GuideStep step={6} title="Run your first sync" tip="On a fresh mapping, most variables show as Changed - that is expected. TokenNexus is comparing Figma's current values against the token values for the first time. Review and apply.">
  Click the **Sync** button. TokenNexus fetches the latest values from GitHub, compares them against your mapped variables, and shows you a diff. Each row shows the variable, the token it maps to, and the before/after values. Select what you want to apply and click **Apply Changes**.

  <ImagePlaceholder label="Screenshot: Sync diff modal showing before and after values for three variables" />
</GuideStep>

<Callout variant="why">
  Sync is intentionally manual. A token update touching hundreds of variables is a meaningful event - it should happen with intent, not silently in the background.
</Callout>
