# Export & Import

TokenNexus can export your complete mapping configuration as a JSON file. This serves as a backup before major changes, a handoff artifact for teammates, and a migration tool when moving a design system to a new Figma file. Importing a mapping file restores the full variable-to-token mapping state, including mode bindings and repository settings.

<ImagePlaceholder label="Screenshot: Export panel showing mapping JSON preview and download button" ratio="16/9" />

## What the export contains

The exported JSON includes the repository URL and branch, the list of selected token files, and every confirmed mapping record - each with the Figma variable ID, variable name, token file path, token key path, and any mode bindings. It does not include your GitHub PAT; you will need to re-enter that after importing into a new file.

## When to export

Export before making large mapping changes you might want to reverse. Export when handing a Figma file to a new team member. Export after completing initial mapping setup on a complex file - treat it as a snapshot of a known-good state.

<Callout variant="tip">
  If your team uses Figma's branching feature, export your mapping whenever you create a new Figma branch. The branch will have its own plugin storage, and importing the mapping means you do not need to re-map from scratch on every branch.
</Callout>
