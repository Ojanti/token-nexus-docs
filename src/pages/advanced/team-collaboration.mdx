# Team Collaboration

TokenNexus stores its mapping in Figma's shared plugin storage, which is accessible to any editor of the file. This means one person can set up the mapping and any other editor with the plugin installed can run syncs - without needing to re-do the mapping work. Each person needs their own GitHub PAT, because credentials are stored per-user, not per-file.

<ImagePlaceholder label="Screenshot: TokenNexus open in a shared Figma file with two active editors" ratio="16/9" />

## How shared storage works

Plugin storage in Figma has two scopes: file-scoped storage (shared across all editors) and user-scoped storage (private to each Figma account). TokenNexus stores the mapping, repository URL, branch, and selected files in file-scoped storage. It stores your GitHub PAT in user-scoped storage. This split is intentional - the mapping is a team asset, but your credentials are personal.

<Callout variant="note">
  Because the mapping is in shared storage, any editor can modify or delete it. Establish a team agreement about who owns the mapping, and use the Export feature to maintain a backup before making significant changes.
</Callout>
