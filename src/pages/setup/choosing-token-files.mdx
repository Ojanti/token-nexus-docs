# Choosing Token Files

After connecting your repository, TokenNexus scans it for JSON files and presents a file tree for you to select from. You choose which files contain the tokens you want to map - this is not necessarily all JSON files in the repository, and it is common to select a subset.

<ImagePlaceholder label="Screenshot: File tree panel showing JSON files with checkboxes for selection" ratio="16/9" />

## How the file scan works

TokenNexus reads the directory tree of your repository via the GitHub API and filters for files with a `.json` extension. It does not fetch the contents of any file until you select it. Files inside `node_modules`, `.git`, and other common non-token directories are excluded automatically.

## Which files to select

Select only the files that define your design tokens. A common pattern is a `tokens/` directory containing one file per category (`colors.json`, `spacing.json`, `typography.json`) or one file per theme (`light.json`, `dark.json`). Do not select build output files, configuration files, or files that import tokens rather than define them.

<Callout variant="tip">
  If you are unsure which files contain your raw token definitions, open them on GitHub first. A token definition file will contain nested objects with `$value` or `value` keys at the leaf nodes. Build output files often contain flattened structures with CSS variable names as keys.
</Callout>
