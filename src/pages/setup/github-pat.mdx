# Generating a GitHub PAT

TokenNexus uses a GitHub Personal Access Token (PAT) to read your token repository. You only need the minimum read scope - no write access is required or requested. The token is stored locally inside Figma's plugin storage and is never sent anywhere other than the GitHub API.

<ImagePlaceholder label="Screenshot: GitHub token creation page with repo:read scope highlighted" ratio="16/9" />

<StepBlock number={1} title="Go to GitHub token settings">
  Navigate to **GitHub** → **Settings** → **Developer settings** → **Personal access tokens** → **Fine-grained tokens**. Fine-grained tokens are recommended because they allow you to scope access to a single repository.
</StepBlock>

<StepBlock number={2} title="Generate a new token">
  Click "Generate new token." Give it a descriptive name like `TokenNexus - [Your Figma File Name]` so you can identify and revoke it later without guessing.
</StepBlock>

<StepBlock number={3} title="Set repository access">
  Under "Repository access," select "Only select repositories" and choose the repository that contains your design tokens. This limits the token to only that repository.
</StepBlock>

<StepBlock number={4} title="Set permissions">
  Under "Repository permissions," set **Contents** to **Read-only**. No other permissions are needed.
</StepBlock>

<StepBlock number={5} title="Set an expiration">
  Choose an expiration that fits your team's security policy. 90 days is a reasonable default. Add a reminder in your calendar to rotate it before it expires - an expired PAT will cause sync to fail with an authentication error.
</StepBlock>

<StepBlock number={6} title="Copy and paste into TokenNexus">
  Copy the generated token immediately - GitHub will not show it again. Paste it into the PAT field in the TokenNexus Setup panel.
</StepBlock>

<Callout variant="warning">
  Treat your PAT like a password. Do not paste it into chat messages, commit it to a repository, or share it in screenshots. If you accidentally expose a token, revoke it immediately from your GitHub token settings and generate a new one.
</Callout>
