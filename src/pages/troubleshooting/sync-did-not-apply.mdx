# Sync Did Not Apply

When a sync session reports success but the Figma variables do not appear to have changed, there are three likely explanations: the variables were already at the target value (no change was needed), the changes were applied to a different variable collection than you were looking at, or a Figma API write error occurred silently. The sync history log is the fastest way to confirm what actually happened.

<ImagePlaceholder label="Screenshot: Sync history panel showing the most recent session with applied variable count" ratio="16/9" />

## The variables were already up to date

If the diff showed changes but the variables already had the target value when you applied, this indicates the Figma variable values were updated between when the diff was calculated and when you clicked Apply. This can happen if another editor applied the same sync, or if you manually edited the variables in Figma between the diff and apply steps. Re-run the diff to confirm the current state.

## Changes applied to a different mode

If your variables have multiple modes and the sync applied to a mode you were not viewing, the changes are present but not visible in the current mode. Switch the variable collection mode in Figma to confirm whether the update was applied to the correct mode.

<Callout variant="warning">
  If the sync history log shows zero variables applied but you confirmed a selection and clicked Apply, this indicates a Figma Plugin API write failure. Check whether the Figma file is in a read-only state - for example, if you are viewing a published library file without edit access.
</Callout>
