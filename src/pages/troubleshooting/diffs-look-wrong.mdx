# Diffs Look Wrong

If the sync diff shows changes you did not expect - variables showing as different when they should match, too many changes, or changes in the wrong direction - do not apply until you understand why. A wrong mapping is far more likely than a bug in the diff engine, so start by verifying the mapping for the variables in question.

<ImagePlaceholder label="Screenshot: Diff panel with a suspicious row highlighted and the token source value circled" ratio="16/9" />

## All variables show as changed after the first sync

This is normal on your first sync if you have not yet synced since creating the mapping. The diff is comparing the current Figma variable values to the current token values, and if the Figma file was created before TokenNexus was installed, the values may never have been in sync. Review each change before applying - this first sync is worth spending time on.

## A color diff shows the wrong values

Check the mapping for that variable. The most common cause is that the variable is mapped to a token that appears to have the right name but is actually in a different category - for example, a background color token mapped to an icon color variable with a similar name. Open the mapping panel, find the variable, and verify that the token path points to the expected location in the JSON file.

<Callout variant="tip">
  Click on any row in the diff to open a detail panel that shows the full token path, the raw JSON value, and the mapping record that produced this diff entry. This is the fastest way to verify whether the diff is correct or whether the mapping needs to be fixed.
</Callout>
