# No Files Found

When the file browser shows an empty list after a successful connection, it means the repository scan did not find any JSON files in the locations it checks, or the files are present but excluded by a built-in filter. This is almost always a path or structure issue, not a permissions issue.

<ImagePlaceholder label="Screenshot: Empty file browser state with empty-state message" ratio="16/9" />

## Token files are not in the repository root

TokenNexus scans the repository root and one level of subdirectories by default. If your token files are nested deeper - for example, in `packages/design-tokens/src/tokens/` - they will not appear in the initial scan. Use the directory path field to navigate to the correct subdirectory before scanning.

## Token files use a non-JSON extension

TokenNexus only lists `.json` files. If your tokens are in `.jsonc`, `.json5`, or `.ts` format, they will not appear in the file browser. The workaround is to ensure your repository also exports a plain JSON version of your tokens.

<Callout variant="note">
  If your repository uses a `tokens.config.js` or `$metadata.json` file to define which token files are active (common in some Style Dictionary setups), TokenNexus does not currently read these configuration files automatically. You will need to select the source token files directly.
</Callout>
