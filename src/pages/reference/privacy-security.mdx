# Privacy & Security

TokenNexus is a local-first plugin. It has no backend server, no analytics, and no account system. Your GitHub Personal Access Token is stored exclusively in Figma's user-scoped plugin storage on your local machine. Your design token data is fetched directly from GitHub to your browser. Nothing passes through a third-party server.

## What is stored and where

| Data | Storage location | Scope | Encrypted |
|------|-----------------|-------|-----------|
| GitHub PAT | Figma user plugin storage | Per user, per Figma account | Yes (Figma's storage is encrypted at rest) |
| Repository URL and branch | Figma file plugin storage | Shared with file editors | No |
| Selected token files | Figma file plugin storage | Shared with file editors | No |
| Mapping records | Figma file plugin storage | Shared with file editors | No |
| Token file contents | In-memory only | Session only, not persisted | N/A |
| Sync history | Figma file plugin storage | Shared with file editors | No |

## What leaves your machine

The only outbound network requests TokenNexus makes are to `api.github.com` (or your GitHub Enterprise Server host). These requests use your PAT as a Bearer token in the Authorization header, and they fetch only the files you have explicitly selected.

<Callout variant="note">
  Figma's plugin sandbox restricts network access. TokenNexus cannot make requests to any domain other than those explicitly listed in its `networkAccess` manifest entry. This is enforced by Figma at the platform level and cannot be bypassed by the plugin code.
</Callout>

## Your PAT and least-privilege access

TokenNexus only needs read access to your repository. A fine-grained PAT with `Contents: read` on a single repository is the minimum viable configuration. We strongly recommend not granting more access than this.
