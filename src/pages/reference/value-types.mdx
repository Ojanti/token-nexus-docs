# Value Type Compatibility

Figma variables have four types: color, number (float), string, and boolean. Design token formats have their own type systems. This reference explains how TokenNexus maps between them, which combinations are compatible, and how values are transformed when types require conversion.

<ImagePlaceholder label="Screenshot: TypeBadge examples in the mapping panel showing all four types" ratio="16/9" />

## Type mapping table

| Token `$type` | Figma Variable Type | Compatible | Notes |
|---------------|---------------------|------------|-------|
| `color` | Color | Yes | Hex, RGB, HSL, and `oklch()` are supported |
| `number` | Number | Yes | Unitless values only; `px` suffix is stripped |
| `dimension` | Number | Partial | The numeric value is extracted; the unit is discarded |
| `string` | String | Yes | Direct value pass-through |
| `boolean` | Boolean | Yes | JSON `true`/`false` maps to Figma boolean |
| `fontFamily` | String | Yes | Stored as a string variable |
| `fontWeight` | Number | Yes | Numeric weight only; named weights (`bold`) require manual mapping |
| `duration` | Number | Partial | Millisecond values are extracted; `ms` suffix is discarded |
| `cubicBezier` | String | Partial | Stored as a string representation |
| `shadow` | - | No | Figma does not have a shadow variable type |
| `gradient` | - | No | Figma does not have a gradient variable type |

## <TypeBadge type="COLOR" /> Color value handling

Color tokens may be expressed as hex (`#3B82F6`), RGB (`rgb(59, 130, 246)`), HSL (`hsl(217, 91%, 60%)`), or the newer `oklch()` format. TokenNexus normalizes all of these to the RGBA representation that the Figma API expects. Alpha channel values are preserved.

## <TypeBadge type="FLOAT" /> Number and dimension handling

Number tokens are passed directly. Dimension tokens (values with a `px`, `rem`, or other CSS unit suffix) have their unit stripped and only the numeric portion is applied to the Figma variable. This means a `dimension` token of `16px` and a `number` token of `16` will produce the same Figma variable value.

<Callout variant="warning">
  If your token file uses `rem` values (for example, `1rem` for 16px at a 16px root), TokenNexus does not automatically convert `rem` to `px`. The raw numeric value `1` will be applied. Ensure your token files use the unit system that matches your Figma variable usage.
</Callout>

## <TypeBadge type="STRING" /> String and <TypeBadge type="BOOLEAN" /> Boolean

String and boolean tokens are passed through exactly as they appear in the token file. No transformation is applied.
